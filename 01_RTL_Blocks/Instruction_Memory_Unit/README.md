Instruction Memory
What is Instruction Memory?

The Instruction Memory stores the program that the processor executes.

Unlike Data Memory, which stores variables and runtime data, Instruction Memory contains the machine instructions generated by the compiler or assembler.

During the Instruction Fetch (IF) stage, the Program Counter (PC) provides an address, and the corresponding 32-bit instruction is fetched from memory.

Why is it Required?

A processor cannot execute instructions unless they are available in memory.

The fetch sequence is straightforward:

Program Counter (PC) ---> Instruction Memory ---> 32-bit Instruction ---> Instruction Decoder

Every instruction executed by the processor begins here.

RTL Operation

Program Initialization

$readmemh("program.hex", Instruction_mem);

Instructions are loaded into memory from a hexadecimal program file before simulation begins.

Instruction Fetch

When

PC = 8

Instruction Memory accesses four consecutive bytes:

Memory[8]
Memory[9]
Memory[10]
Memory[11]

These bytes are combined into a single 32-bit instruction:

Instruction_Out =
{
Memory[11],
Memory[10],
Memory[9],
Memory[8]
}

For example:

Instruction = 0x00A00093

which corresponds to:

ADDI x1, x0, 10
Verification Strategy

The module was verified by:

Loading instructions into memory
Fetching instructions using different Program Counter values
Reconstructing 32-bit instructions from byte-addressable memory
Comparing fetched instructions with expected machine code

Simulation confirmed correct instruction reconstruction and instruction fetch behavior.

Design Decisions:

Byte-addressable memory organization to match the RISC-V ISA.

Separate Instruction Memory and Data Memory, following a Harvard-style architecture in this implementation.

Program initialization through $readmemh for flexibility during simulation and testing.

Combinational instruction fetch, appropriate for the current single-cycle processor design.**
