Immediate Extend Unit
What is the Immediate Extend Unit?

The Immediate Extend Unit extracts the immediate field from a RISC-V instruction and converts it into a 32-bit value.

Since RISC-V uses different immediate layouts for different instruction formats, this module reconstructs the correct value before it is used by the datapath.

Why is it Required?

Many RISC-V instructions depend on immediate values.

These include:

Arithmetic Immediate Instructions
Load Instructions
Store Instructions
Branch Instructions
Jump Instructions
Upper Immediate Instructions

Without this module, the ALU and branch logic would receive incorrect operands.

RTL Features
Pure combinational logic
Immediate extraction based on instruction format
Sign extension for I, S, B, and J formats
Upper immediate generation for U-Type instructions
32-bit immediate output
Interface
Signal	Width	Description
instr_in	32	RISC-V instruction
Imm_type_in	3	Immediate format selector
Imm_out	32	Sign-extended immediate value
RTL Operation

The module selects the extraction logic using the Imm_type_in control signal generated by the Instruction Decoder.

Depending on the instruction format, it reconstructs the immediate field and produces a 32-bit output for downstream modules such as the ALU and Immediate Adder.
---
Verification Strategy

The module was verified using dedicated test cases for:

I-Type immediate
S-Type immediate
B-Type immediate
U-Type immediate
J-Type immediate

Each test confirmed that the extracted immediate matched the expected 32-bit value.
