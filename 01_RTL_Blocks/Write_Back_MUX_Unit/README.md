
Write-Back Multiplexer (WB MUX)
What is the Write-Back Multiplexer?

The Write-Back Multiplexer (WB MUX) selects the final result that will be written into the Register File.

Different RISC-V instructions produce results from different parts of the datapath. Rather than connecting every source directly to the Register File, the WB MUX chooses the correct value based on the control signals generated by the Instruction Decoder.

This is the final stage of the datapath before the processor completes an instruction.

Why is it Required?

Different instruction classes generate results from different hardware blocks:

Instruction Type	Write-Back Source
R-Type / I-Type ALU	ALU Result
Load	Data Memory
JAL / JALR	PC + 4
LUI	Immediate Value
AUIPC	PC + Immediate

The WB MUX ensures that only the correct result reaches the Register File.

Design Decisions
Centralized write-back selection simplifies datapath integration.
Combinational implementation avoids additional clock latency.
Decoder exclusively controls source selection, keeping datapath logic modular and scalable.
